Crie um novo projeto

O Upstash fornece um serviço de Redis gratuito. O Redis é usado pelo ClaudeZap para controlar a fila de mensagens — garantindo que as mensagens sejam enviadas com o delay certo (respeitando os limites do WhatsApp). O plano gratuito oferece 10.000 operações por dia.

Nome: claudezap

A URL correta começa com rediss:// (com dois "s" no final). Esse "s" extra indica que a conexão é segura (criptografada). Se começar com apenas redis://, busque outra opção de conexão no painel.

Armazenamento de Arquivos — Cloudflare R2
Onde as imagens, PDFs e documentos das campanhas ficam guardados  ·  ⏱ ~8 min
🆕
Novidade na versão 2.0
O ClaudeZap agora permite enviar imagens, PDFs, documentos Word, planilhas e muito mais nas campanhas. Para isso, os arquivos precisam de um lugar na internet para ficarem guardados. O Cloudflare R2 é esse lugar — gratuito para os nossos volumes de uso.
ℹ️
O que é o Cloudflare R2?
O R2 é um serviço de armazenamento de arquivos em nuvem da Cloudflare. Pense nele como um "Google Drive" que o sistema usa para guardar os arquivos antes de enviá-los pelo WhatsApp. O plano gratuito inclui 10 GB de espaço e 1 milhão de operações por mês — mais que suficiente para uso normal. E o melhor: sem custo de transferência de dados.
1
Crie uma conta na Cloudflare
Acesse o site e crie uma conta gratuita:
https://dash.cloudflare.com/sign-up
a)
Preencha e-mail e senha e clique em "Create Account"
b)
Confirme o e-mail clicando no link que a Cloudflare enviar
c)
Faça login no painel em dash.cloudflare.com
2
Ative o R2 Object Storage
a)
No painel da Cloudflare, clique em "R2 Object Storage" no menu lateral esquerdo
b)
Se aparecer uma tela de ativação, clique em "Purchase R2 Plan" — não se preocupe, o plano gratuito é selecionado automaticamente, sem custo
c)
Pode ser necessário adicionar um cartão de crédito como garantia (a Cloudflare não cobra nada enquanto você estiver dentro do plano gratuito)
3
Crie o bucket (pasta de armazenamento)
a)
Na tela do R2, clique em "Create bucket"
b)
Nome do bucket: claudezap-media (exatamente assim, em minúsculas com hífen)
c)
Location: deixe em "Automatic"
d)
Clique em "Create bucket"
4
Ative o acesso público ao bucket
Para que as mensagens do WhatsApp consigam exibir os arquivos, o bucket precisa ser público.
a)
Clique no bucket claudezap-media para abri-lo
b)
Clique na aba "Settings"
c)
Role até a seção "Public access"
d)
Clique em "Allow Access" e confirme. Isso vai gerar uma URL pública no formato pub-XXXXX.r2.dev — anote essa URL com o rótulo R2_PUBLIC_URL
5
Anote o Account ID da Cloudflare
Você vai precisar do ID da sua conta Cloudflare. Ele fica visível na URL do painel e também na página inicial do R2.
a)
Olhe a URL do navegador enquanto está no painel. Ela tem o formato: dash.cloudflare.com/SEU_ACCOUNT_ID/r2/...
b)
Copie essa sequência de letras e números (é um código hexadecimal de 32 caracteres, como: a1b2c3d4e5f6...)
c)
Anote no bloco de notas com o rótulo R2_ACCOUNT_ID
6
Crie um token de API do R2
Agora vamos criar as "chaves" que permitem o sistema fazer upload de arquivos.
a)
Na tela principal do R2, clique em "Manage R2 API Tokens" (canto superior direito)
b)
Clique em "Create API token"
c)
Nome do token: claudezap-token
d)
Permissões: selecione "Object Read & Write"
e)
Specify bucket (opcional): selecione claudezap-media para restringir apenas a este bucket
f)
Clique em "Create API Token"
7
Copie o Access Key ID e o Secret Access Key
Após criar o token, a Cloudflare vai mostrar uma única vez as duas chaves. Copie agora — depois não será possível ver o Secret novamente.
a)
Copie o Access Key ID (uma sequência de 32 letras e números) — anote com o rótulo R2_ACCESS_KEY_ID
b)
Copie o Secret Access Key — anote com o rótulo R2_SECRET_ACCESS_KEY
c)
Clique em "Finish" apenas depois de ter copiado os dois valores
⚠️
ATENÇÃO: o Secret Access Key aparece só uma vez!
Depois de fechar essa tela, a Cloudflare não mostra mais o Secret Access Key. Se perder, será necessário deletar o token e criar um novo. Não feche a página antes de copiar os dois valores.
✅
Resumo: você deve ter anotado 5 valores do R2
R2_ACCOUNT_ID — ID da conta Cloudflare (32 chars hex)
R2_ACCESS_KEY_ID — Chave de acesso do token (32 chars)
R2_SECRET_ACCESS_KEY — Senha secreta do token
R2_BUCKET — nome do bucket: claudezap-media
R2_PUBLIC_URL — URL pública do bucket (ex: https://pub-abc123.r2.dev)

Como usar as novas funcionalidades
Guia prático para aproveitar ao máximo o ClaudeZap v2.0
📦
Envio em Lotes Anti-Bloqueio

O WhatsApp pode restringir números que enviam muitas mensagens de uma vez para contatos diferentes. O modo de envio em lotes divide sua campanha em grupos menores com um intervalo de horas entre cada grupo, imitando um comportamento mais natural e reduzindo drasticamente o risco de bloqueio.

1.
Na criação de uma campanha, role até a seção "Configurações de Envio"
2.
Ative a chave "Envio em Lotes"
3.
Defina o Tamanho do lote — recomendamos entre 30 e 50 mensagens por lote
4.
Defina o Intervalo entre lotes — recomendamos 8 horas para máxima segurança (mínimo 1 hora)
5.
Crie a campanha. O sistema vai enviar o 1º lote imediatamente, depois aguardar o intervalo configurado antes do 2º lote, e assim por diante
💡
Exemplo prático
Para enviar para 200 contatos com lotes de 50 e intervalo de 8h: o sistema vai enviar 50 mensagens agora, mais 50 daqui a 8h, mais 50 daqui a 16h, e os últimos 50 daqui a 24h. Muito mais seguro do que enviar tudo de uma vez!

Rastreamento de Contactados

O sistema agora registra automaticamente quando um contato foi incluído em uma campanha. Isso permite filtrar e evitar enviar para as mesmas pessoas repetidamente, o que reduz reclamações e o risco de bloqueio.

1.
Na lista de contatos, contatos que já receberam uma campanha mostram um badge "Enviado" em verde ao lado do nome
2.
Use o filtro "Apenas não contactados" para ver somente quem ainda não recebeu nenhuma campanha
3.
Ao criar uma nova campanha, ative a opção "Excluir já contactados" — o sistema vai automaticamente pular os contatos que já foram alcançados por campanhas anteriores
4.
Para reiniciar o rastreamento de um contato (quero enviar novamente para ele), acesse o contato e clique em "Limpar histórico de campanhas"
💡
Estratégia recomendada
Importe toda a sua lista de contatos e use "Excluir já contactados" em todas as campanhas. Assim cada pessoa recebe a mensagem uma única vez, o sistema vai gradualmente "consumindo" a lista sem repetições.

Melhorias de Segurança (v2.0)

A versão 2.0 inclui várias melhorias de segurança transparentes ao usuário. Você não precisa configurar nada — elas funcionam automaticamente.

Bloqueio por tentativas de login: Após 5 tentativas erradas de senha, a conta é bloqueada por 30 minutos automaticamente. Protege contra ataques de força bruta.
Proteção de origem (CORS): O servidor só aceita requisições vindas do seu domínio. Nenhum outro site pode usar sua API.
Planos verificados em cada requisição: Usuários com trial expirado ou conta inativa são bloqueados automaticamente em qualquer operação.
Limite de tamanho de requisições: Requests são limitados a 1 MB para evitar ataques de sobrecarga.
Rate limiting no formulário de opt-in: O link de cadastro público é limitado a 20 submissões/hora por usuário, evitando spam na sua lista de contatos.
