Dica importante antes de começar
Abra um bloco de notas (ou um arquivo Word) para ir anotando as informações que você vai copiar ao longo do processo — como senhas, URLs e chaves. Isso vai facilitar muito o processo.
1
Criar conta no GitHub
Onde o código do sistema fica armazenado  ·  ⏱ ~3 min
ℹ️
O que é o GitHub?
O GitHub é uma plataforma onde o código do ClaudeZap está armazenado. Você vai conectar sua conta do GitHub nas outras plataformas (Vercel, Render) para que elas possam buscar o código e colocar no ar automaticamente.
1
Acesse o GitHub
Abra seu navegador e acesse:
https://github.com
2
Crie sua conta
Clique em "Sign up" (canto superior direito) e preencha os dados solicitados.
a)
Digite seu endereço de e-mail
b)
Crie uma senha (mínimo 8 caracteres, com letras e números)
c)
Escolha um nome de usuário (ex: clicamkt)
d)
Complete a verificação "I'm not a robot"
e)
Clique em "Create account"
f)
Verifique seu e-mail e clique no link de confirmação
3
Acesse o repositório do ClaudeZap
Com a conta criada e logada, acesse o repositório do sistema:
https://github.com/clicasenderzap-wq/claudezap
Clique no botão "Fork" (canto superior direito da página) para copiar o projeto para a sua conta. Confirme clicando em "Create fork".
✅
Como saber que deu certo?
Após o fork, você será redirecionado para uma cópia do repositório na sua conta, com o endereço: https://github.com/SEU-USUARIO/claudezap
⚠️
Anote no seu bloco de notas
Guarde a URL do seu repositório fork. Ex: https://github.com/clicamkt/claudezap
2
Banco de Dados — Neon
Onde os contatos e mensagens ficam armazenados  ·  ⏱ ~5 min
ℹ️
O que é o Neon?
O Neon é um serviço de banco de dados gratuito baseado em PostgreSQL. É onde o ClaudeZap vai salvar seus contatos, histórico de mensagens e informações de usuário. O plano gratuito oferece 0.5 GB de armazenamento — suficiente para milhares de contatos.
1
Acesse o Neon
https://neon.tech
2
Crie sua conta
Clique em "Sign up" e escolha entrar com o GitHub (mais rápido — já tem conta!).
a)
Clique em "Continue with GitHub"
b)
Autorize o Neon a acessar sua conta GitHub
c)
Preencha as perguntas iniciais (pode pular ou escolher qualquer opção)
3
Crie um novo projeto
a)
Clique em "Create a project"
b)
Nome do projeto: claudezap
c)
Database name: claudezap
d)
Region: escolha "AWS / São Paulo" se disponível, senão "AWS / US East"
e)
Clique em "Create project"
4
Copie a Connection String (DATABASE_URL)
Após criar o projeto, o Neon vai mostrar uma tela de conexão. Procure o campo chamado "Connection string".
a)
Certifique-se que a opção "Pooled connection" está marcada
b)
Clique no ícone de copiar ao lado da connection string
c)
Cole no seu bloco de notas com o rótulo DATABASE_URL
# Exemplo de como vai parecer (seus dados serão diferentes): DATABASE_URL=postgresql://claudezap_owner:AbCdEfGh@ep-cold-rain-a5b1c2d3-pooler.us-east-2.aws.neon.tech/claudezap?sslmode=require
✅
Como saber que deu certo?
Você verá o painel do Neon com o banco claudezap criado e a connection string disponível. Guarde essa string — ela é a "senha de acesso" ao banco de dados.
3
Cache — Upstash (Redis)
Gerencia a fila de envio de mensagens  ·  ⏱ ~5 min
ℹ️
O que é o Upstash?
O Upstash fornece um serviço de Redis gratuito. O Redis é usado pelo ClaudeZap para controlar a fila de mensagens — garantindo que as mensagens sejam enviadas com o delay certo (respeitando os limites do WhatsApp). O plano gratuito oferece 10.000 operações por dia.
1
Acesse o Upstash
https://upstash.com
2
Crie sua conta
a)
Clique em "Sign up"
b)
Escolha "Continue with GitHub"
c)
Autorize o Upstash
3
Crie um banco Redis
a)
No painel, clique em "Create database"
b)
Escolha o tipo "Redis"
c)
Nome: claudezap
d)
Region: "São Paulo, Brazil" se disponível
e)
Plano: "Free"
f)
Clique em "Create"
4
Copie a Redis URL
a)
Clique no banco criado para abrir os detalhes
b)
Role até a seção "Connect to your database"
c)
Procure o campo "UPSTASH_REDIS_REST_URL" — não! Procure a aba "Redis CLI" ou o campo chamado "Endpoint"
d)
Na seção "Connect your client", selecione "ioredis" ou copie a URL no campo REDIS_URL (começa com rediss://)
e)
Cole no bloco de notas com o rótulo REDIS_URL
# Exemplo de como vai parecer: REDIS_URL=rediss://default:AaBbCcDd@us1-warm-robin-12345.upstash.io:6380
⚠️
Atenção com a URL do Redis
A URL correta começa com rediss:// (com dois "s" no final). Esse "s" extra indica que a conexão é segura (criptografada). Se começar com apenas redis://, busque outra opção de conexão no painel.
4
Backend — Render
Servidor que processa os envios e a lógica do sistema  ·  ⏱ ~10 min
ℹ️
O que é o Render?
O Render hospeda o servidor (backend) do ClaudeZap — a parte que processa os envios, autentica usuários e se comunica com o WhatsApp. O plano gratuito suporta até 750 horas de uso por mês. Atenção: no plano gratuito, o servidor fica inativo após 15 minutos sem uso e demora ~30 segundos para "acordar" na primeira requisição.
1
Acesse o Render
https://render.com
2
Crie sua conta
a)
Clique em "Get Started for Free"
b)
Escolha "Continue with GitHub"
c)
Autorize o Render
3
Crie um novo Web Service
a)
No painel, clique em "New +" → "Web Service"
b)
Selecione "Build and deploy from a Git repository"
c)
Clique em "Connect GitHub" e autorize o acesso
d)
Procure e selecione o repositório claudezap
4
Configure o serviço
Preencha os campos conforme abaixo:
CampoValor a preencher
Nameclaudezap-api
RegionOregon (US West) — padrão
Branchmain
Root Directorybackend
RuntimeNode
Build Commandnpm install
Start Commandnode src/app.js
Instance TypeFree
5
Adicione as variáveis de ambiente
Role a página até a seção "Environment Variables" e adicione cada linha clicando em "Add Environment Variable":
Nome da variávelValor
NODE_ENVproduction
JWT_EXPIRES_IN7d
MSG_DELAY_MS3000
MSG_RATE_LIMIT_PER_MIN20
WA_SESSION_DIR./wa_sessions
DATABASE_URLCole a string do Neon (passo 2)
REDIS_URLCole a URL do Upstash (passo 3)
CORS_ORIGINDeixe vazio por enquanto — preencher no passo 5
⚠️
Sobre o JWT_SECRET
O Render vai gerar o JWT_SECRET automaticamente por conta do render.yaml. Você não precisa criar esse manualmente.
6
Inicie o deploy
a)
Clique em "Create Web Service"
b)
Aguarde o build finalizar (3–5 minutos). Você verá logs em tempo real na tela
c)
Quando aparecer Live verde no topo, o servidor está no ar
7
Anote a URL do backend
No topo da página, você verá a URL do seu servidor. Ela será algo como:
https://claudezap-api.onrender.com
Anote essa URL no bloco de notas com o rótulo URL_BACKEND. Você vai precisar dela no próximo passo.
8
Teste se o servidor está funcionando
Abra uma nova aba no navegador e acesse:
https://claudezap-api.onrender.com/health
Deve aparecer na tela: {"status":"ok","timestamp":"..."}
5
Frontend — Vercel
A tela visual que você vai usar no dia a dia  ·  ⏱ ~8 min
ℹ️
O que é o Vercel?
O Vercel hospeda o frontend (a interface visual) do ClaudeZap. É a tela que você vai acessar pelo navegador para enviar mensagens, gerenciar contatos e ver o dashboard. O plano gratuito é generoso e mais que suficiente.
1
Acesse o Vercel
https://vercel.com
2
Crie sua conta
a)
Clique em "Sign Up"
b)
Escolha "Continue with GitHub"
c)
Autorize o Vercel
d)
Quando perguntar sobre plano, escolha "Hobby" (gratuito)
3
Importe o projeto
a)
No painel, clique em "Add New → Project"
b)
Selecione o repositório claudezap da lista
c)
Clique em "Import"
4
Configure o projeto
O Vercel vai detectar automaticamente o arquivo vercel.json e já vai saber que o frontend está na pasta frontend/. Verifique se está correto:
CampoValor esperado
Framework PresetNext.js
Root Directoryfrontend
Build Commandnpm run build
Output Directory.next
5
Adicione a variável de ambiente
Ainda na tela de configuração, abra a seção "Environment Variables" e adicione:
NomeValor
NEXT_PUBLIC_API_URLURL do Render + /api
Ex: https://claudezap-api.onrender.com/api
6
Faça o deploy
a)
Clique em "Deploy"
b)
Aguarde o build (2–4 minutos)
c)
Quando aparecer "Congratulations!" com confetes, o frontend está no ar
7
Anote a URL do frontend
Clique em "Continue to Dashboard". A URL do seu site será algo como:
https://claudezap.vercel.app
Anote com o rótulo URL_FRONTEND.
8
Atualize o CORS no Render
Agora você já tem a URL do frontend. Volte ao painel do Render e atualize a variável CORS_ORIGIN:
a)
No painel do Render, clique no serviço claudezap-api
b)
Vá em "Environment" no menu lateral
c)
Encontre CORS_ORIGIN e preencha com a URL do Vercel (ex: https://claudezap.vercel.app)
d)
Clique em "Save Changes" — o servidor vai reiniciar automaticamente
6
Conectar Domínio — clicaai.ai.br
Apontar seu domínio para o Vercel e Render via Hostinger  ·  ⏱ ~10 min
ℹ️
Como funciona?
Você vai configurar o DNS do seu domínio na Hostinger para que:
• clicaai.ai.br (sem prefixo) aponte para o Vercel (frontend)
• api.clicaai.ai.br aponte para o Render (backend)
Parte A — Adicionar domínio no Vercel
1
Acesse as configurações do projeto no Vercel
a)
No painel do Vercel, clique no projeto claudezap
b)
Clique em "Settings" no menu superior
c)
Clique em "Domains" no menu lateral
2
Adicione o domínio
a)
Digite clicaai.ai.br no campo e clique em "Add"
b)
O Vercel vai mostrar os registros DNS que você precisa configurar
c)
Anote os valores — vão aparecer como na tabela abaixo
TipoNomeValor (exemplo)
A@76.76.21.21
CNAMEwwwcname.vercel-dns.com
Parte B — Adicionar domínio no Render
3
Configure o domínio da API no Render
a)
No painel do Render, clique no serviço claudezap-api
b)
Vá em "Settings" → "Custom Domains"
c)
Clique em "Add Custom Domain"
d)
Digite: api.clicaai.ai.br e confirme
e)
Anote o valor CNAME que o Render mostrar (algo como claudezap-api.onrender.com)
Parte C — Configurar DNS na Hostinger
4
Acesse o painel da Hostinger
https://hpanel.hostinger.com
a)
Faça login com suas credenciais da Hostinger
b)
Clique em "Domínios" no menu
c)
Encontre clicaai.ai.br e clique em "Gerenciar"
d)
Clique em "DNS / Nameservers"
5
Adicione os registros DNS
Para cada linha abaixo, clique em "Adicionar registro" e preencha os campos:
TipoNome / HostValor / Aponta paraTTL
A@IP fornecido pelo Vercel
(ex: 76.76.21.21)		3600
CNAMEwwwcname.vercel-dns.com3600
CNAMEapiURL do Render
(ex: claudezap-api.onrender.com)		3600
6
Aguarde a propagação do DNS
Após salvar os registros, aguarde entre 5 minutos e 24 horas para o DNS propagar. Na maioria dos casos acontece em menos de 30 minutos.
✅
Como verificar se o DNS propagou?
Acesse https://dnschecker.org e pesquise por clicaai.ai.br. Quando aparecerem bolinhas verdes na maioria dos servidores, o DNS já propagou.
⚠️
Atualize o CORS_ORIGIN no Render
Após o domínio funcionar, volte ao Render e atualize CORS_ORIGIN de https://claudezap.vercel.app para https://clicaai.ai.br.
7
Conectar o WhatsApp via QR Code
Vincular seu número ao sistema  ·  ⏱ ~3 min
⚠️
Use um número exclusivo para disparos
Recomendamos fortemente usar um número de WhatsApp dedicado para os disparos — não o seu número pessoal. Isso evita que seu número principal seja bloqueado pelo WhatsApp.
1
Acesse o ClaudeZap
Abra o navegador e acesse seu site:
https://clicaai.ai.br
(ou a URL do Vercel se ainda não configurou o domínio)
2
Crie sua conta no sistema
a)
Clique em "Cadastre-se"
b)
Preencha nome, e-mail e senha
c)
Clique em "Criar conta"
3
Vá para a página WhatsApp
a)
No menu lateral, clique em "WhatsApp"
b)
Clique no botão "Conectar WhatsApp"
c)
Um QR Code vai aparecer na tela — aguarde
4
Escaneie o QR Code com o celular
a)
No celular, abra o WhatsApp
b)
Toque nos três pontinhos (Android) ou "Configurações" (iPhone)
c)
Selecione "Dispositivos conectados"
d)
Toque em "Conectar um dispositivo"
e)
Aponte a câmera para o QR Code na tela do computador
5
Confirmação de conexão
Após escanear, a página vai atualizar e mostrar o status como "Conectado" com um ícone verde. Seu WhatsApp está vinculado ao sistema!
⚠️
Lembrete importante — plano gratuito do Render
No plano gratuito do Render, o servidor "dorme" após 15 minutos sem uso. Quando isso acontece, a sessão do WhatsApp é perdida e você precisará escanear o QR Code novamente. Para uso em produção real, considere fazer o upgrade do Render ($7/mês) que mantém o servidor sempre ativo.
8
Teste Final e Validação
Verificar se tudo está funcionando corretamente  ·  ⏱ ~5 min
1
Adicione um contato de teste
a)
No menu lateral, clique em "Contatos"
b)
Clique em "Novo contato"
c)
Preencha com seus próprios dados (name + seu número de celular)
d)
O telefone deve ter o formato internacional: 5511999999999 (55 = Brasil, 11 = DDD, + número)
e)
Clique em "Salvar"
2
Envie uma mensagem de teste
a)
Clique em "Mensagens" no menu
b)
Selecione o contato que você criou
c)
Escreva uma mensagem: Olá {{nome}}, este é um teste do ClaudeZap!
d)
Clique em "Enviar"
3
Verifique o recebimento
Em alguns segundos você deverá receber a mensagem no seu celular via WhatsApp. A mensagem vai aparecer com o nome substituído: "Olá João, este é um teste do ClaudeZap!"
4
Verifique o histórico
Na própria página de Mensagens, role a tela para baixo e veja o Histórico. A mensagem enviada deve aparecer com o status "Enviado" em verde.
5
Verifique o Dashboard
Clique em "Dashboard" no menu. Você vai ver os contadores atualizados com os dados do envio de teste.
🎉
Parabéns! O sistema está no ar!
Se a mensagem chegou e o histórico mostra "Enviado", seu ClaudeZap está funcionando perfeitamente. Agora você pode começar a importar seus contatos e criar campanhas de envio.
